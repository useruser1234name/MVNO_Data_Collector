# 수집 법적·윤리 가드레일

40+ 사업자 및 이통3사로 확장 시 차단·법적 리스크가 비선형으로 증가한다.
지속가능한(=차단당하지 않는) 수집을 위해 아래 원칙을 코드와 운영에 강제한다.

## 원칙
1. **robots.txt 준수 기본값** — `collectors/utils/robots.py`의 `RobotsCache`로 검사.
   요금제는 공개 마케팅 정보이나, 명시적 차단 경로는 수집하지 않는다.
2. **식별 가능한 User-Agent** — 연락처 포함 UA(`IDENTIFIABLE_USER_AGENT`)를 옵션 제공.
   상대가 문제 시 연락·합의할 수 있게 한다.
3. **크롤링 예산** — 도메인별 일일 요청 상한(`CrawlBudget`)으로 과부하 방지.
   동일 호스팅/솔루션 공유 사업자에 대한 동시 부하는 도메인 레이트리미터로 통제.
4. **캡차/봇차단 우회 지양** — 차단 감지(`looks_blocked`) 시 우회가 아니라
   백오프·요청감속·운영자 알림·합의를 우선한다.
5. **PII 무수집** — 개인식별정보는 수집·저장하지 않는다. 공개 요금제 정보만 대상.
6. **이통3사 보수적 프로파일** — `anti_bot_level: high` → 저빈도 스케줄, 낮은 동시성,
   지수 백오프, 전용 풀(`mno_pool`). 사업자별 ToS를 사전 검토.

## 운영 체크리스트
- [ ] 신규 사업자 robots.txt 확인 및 차단 경로 회피
- [ ] anti_bot_level에 맞는 스케줄/동시성 설정(카탈로그)
- [ ] 차단 발생 시 즉시 감속·알림, 필요 시 사업자 연락
- [ ] 법무 검토가 필요한 사업자는 enabled=false로 보류
